# แผนสร้างรายได้ — Expense Tracking App (B2C)

> เอกสารวางโครงสร้าง Pricing Tier และ Feature Roadmap สำหรับเปลี่ยนแอปบันทึกรายจ่ายส่วนตัวให้เก็บเงินได้จริง
> โฟกัสตลาด: **ผู้ใช้ทั่วไป (B2C) ในไทย**
> อัปเดต: 31 พ.ค. 2026

---

## 1. สรุปสถานะปัจจุบันของโปรเจค

แอปปัจจุบันมีฐานที่แข็งแรงพอจะต่อยอดเชิงพาณิชย์ได้ทันที:

- **Stack:** SvelteKit + Prisma + PostgreSQL + Auth.js (รองรับ multi-user แล้ว)
- **ฟีเจอร์ที่มี:** บันทึกรายจ่าย, หมวดหมู่ (category) พร้อมงบต่อเดือน (monthly limit), dashboard สรุปยอด, item prefix ช่วยกรอกเร็ว
- **ช่องว่างที่ต้องเติมเพื่อเก็บเงิน:** ยังไม่มี import ธนาคาร, OCR, รายงานเชิงลึก, export, multi-currency, recurring, การจำกัดสิทธิ์ตาม tier

จุดแข็งที่ใช้เป็นหมัดเด็ดในตลาดไทย: **การ import statement ธนาคารไทย / PromptPay** ซึ่งคู่แข่งสากลทำได้ไม่ดี

## 2. Benchmark ราคาคู่แข่ง (อ้างอิงจริง)

อัตราแปลงโดยประมาณ 1 USD ≈ 36 THB

| แอป | ราคา/เดือน | ราคา/ปี | หมายเหตุ |
|---|---|---|---|
| **Money Lover** | ~79 ฿ (Budget Plus) | ~569 ฿ Premium / ~749 ฿ Linked Wallet | เชื่อมบัญชีธนาคารไทยอยู่ใน tier บนสุด |
| **Spendee** | $2.99–5.99 (~108–215 ฿) | $22.99–35.99 (~830–1,300 ฿) | มี Plus และ Premium สองชั้น |
| **YNAB** | $14.99 (~540 ฿) | $109 (~3,900 ฿) | ราคาพรีเมียม ไม่มี free tier, เน้น zero-based budgeting |

**ข้อสรุปเชิงราคา:** ตลาดไทยคุ้นกับช่วง **40–120 ฿/เดือน** หรือ **400–900 ฿/ปี** สำหรับแอปการเงินส่วนบุคคล การตั้งราคาในกรอบนี้จึงปลอดภัยและแข่งขันได้ ส่วน YNAB เป็นกลุ่มพรีเมียมที่คนไทยทั่วไปมองว่าแพง — ใช้เป็นเพดานบน ไม่ใช่เป้า

## 3. โครงสร้าง Pricing Tier ที่แนะนำ

ใช้โมเดล **Freemium 3 ชั้น** — Free ดึงคน, Plus เป็นตัวทำเงินหลัก, Pro เก็บกลุ่มจริงจัง

### 🆓 Free — "เริ่มต้น"
**ราคา: 0 ฿**
เป้าหมาย: ให้คนเข้ามาใช้ ติดเป็นนิสัย และเห็นคุณค่าก่อนชนกำแพง

- บันทึกรายจ่ายไม่จำกัดจำนวนรายการ
- หมวดหมู่สูงสุด **5 หมวด**
- ตั้งงบต่อเดือนได้ **1 งบ**
- Dashboard พื้นฐาน (เดือนปัจจุบัน)
- ดูประวัติย้อนหลัง **3 เดือน**
- 1 บัญชี/กระเป๋า (wallet)

> **กำแพงที่ตั้งใจวาง:** จำกัดหมวด, ประวัติย้อนหลัง, และจำนวนบัญชี — สามจุดนี้คือแรงผลักให้อัปเกรด โดยไม่ทำให้แอปใช้ไม่ได้

### ⭐ Plus — "ตัวจริง" (Tier ทำเงินหลัก)
**ราคา: 59 ฿/เดือน หรือ 499 ฿/ปี** (ประหยัด ~30% เมื่อจ่ายรายปี)
เป้าหมาย: ผู้ใช้ที่จริงจังกับการคุมเงินส่วนตัว

ได้ทุกอย่างใน Free บวก:

- หมวดหมู่ **ไม่จำกัด** + งบไม่จำกัด
- ประวัติย้อนหลัง **ไม่จำกัด**
- หลายบัญชี/กระเป๋า (เงินสด, ธนาคาร, บัตร)
- รายจ่ายแบบ **recurring** (ค่าเช่า, ค่าสมาชิก อัตโนมัติ)
- **รายงานเชิงลึก** — เทรนด์รายเดือน/ปี, แยกตามหมวด, เปรียบเทียบช่วงเวลา
- **Export** เป็น Excel / CSV / PDF
- แนบรูปใบเสร็จ
- สำรอง & กู้คืนข้อมูล

### 🚀 Pro — "ครบเครื่อง"
**ราคา: 99 ฿/เดือน หรือ 899 ฿/ปี**
เป้าหมาย: ผู้ใช้ที่ต้องการระบบอัตโนมัติเต็มรูปแบบ

ได้ทุกอย่างใน Plus บวก:

- **Import statement ธนาคารไทย / PromptPay อัตโนมัติ** ← หมัดเด็ด
- **OCR สแกนใบเสร็จ** กรอกอัตโนมัติ
- **จัดหมวดหมู่ด้วย AI** อัตโนมัติ
- รายงานภาษี / สรุปสำหรับยื่นภาษี
- เป้าหมายการออม (savings goals) + แจ้งเตือนอัจฉริยะ
- ความสำคัญในการ support

### ตารางเปรียบเทียบ

| ฟีเจอร์ | Free | Plus | Pro |
|---|:---:|:---:|:---:|
| บันทึกรายจ่าย | ✓ ไม่จำกัด | ✓ | ✓ |
| หมวดหมู่ | 5 | ไม่จำกัด | ไม่จำกัด |
| ประวัติย้อนหลัง | 3 เดือน | ไม่จำกัด | ไม่จำกัด |
| จำนวนบัญชี/กระเป๋า | 1 | หลายบัญชี | หลายบัญชี |
| Recurring | – | ✓ | ✓ |
| รายงานเชิงลึก | – | ✓ | ✓ |
| Export (Excel/PDF) | – | ✓ | ✓ |
| แนบใบเสร็จ | – | ✓ | ✓ |
| Import ธนาคาร/PromptPay | – | – | ✓ |
| OCR ใบเสร็จ | – | – | ✓ |
| AI จัดหมวด | – | – | ✓ |
| รายงานภาษี | – | – | ✓ |
| **ราคา/เดือน** | 0 ฿ | 59 ฿ | 99 ฿ |
| **ราคา/ปี** | 0 ฿ | 499 ฿ | 899 ฿ |

## 4. หลักจิตวิทยาและกลยุทธ์ราคาที่ใช้

- **Anchor ที่ Pro:** ตั้ง Pro ไว้สูงเพื่อให้ Plus ดู "คุ้มที่สุด" — คนส่วนใหญ่จะเลือกตัวกลาง
- **Annual discount ~30%:** ดันให้จ่ายรายปี → กระแสเงินสดล่วงหน้า + ลด churn
- **กำแพง 3 จุดใน Free** (หมวด/ประวัติ/บัญชี) จูงใจอัปเกรดโดยไม่ทำให้แอปดูใช้ไม่ได้
- **ราคา 59/99 ฿** อยู่ใต้เพดานจิตวิทยา 100 ฿ และต่ำกว่า Money Lover เล็กน้อย → เข้าได้ง่าย
- **7-day free trial ของ Plus/Pro** ให้ลองฟีเจอร์เด่นก่อนตัดสินใจ
- เริ่มราคาintroductory ต่ำได้ในช่วงเปิดตัว (เช่น 39 ฿) แล้วค่อยขยับ — แต่ล็อกราคาเดิมให้ผู้ใช้ยุคแรก (grandfather pricing) เพื่อสร้าง goodwill

## 5. Feature Roadmap — เรียงตามผลต่อรายได้

วาง roadmap แบบ 4 phase โดยจัดลำดับจาก "ทำง่าย-ผลเร็ว" ไปหา "ทำยาก-มูลค่าสูง"

### Phase 0 — โครงสร้างพื้นฐานเก็บเงิน (จำเป็นก่อนเปิดขาย)
*ไม่มีอันนี้ = เก็บเงินไม่ได้*

> 🔄 **โมเดลที่ใช้จริงถูกปรับเป็น Progressive Trial Funnel** (ทดลอง 3+4 วัน → เช่า/รายเดือน) — ดู [`phase-0-trial-funnel-spec.md`](./phase-0-trial-funnel-spec.md)
> กลไก billing เบื้องหลังดูใน [`phase-0-billing-spec.md`](./phase-0-billing-spec.md)

- ระบบ **subscription & billing** (เชื่อม Stripe / Omise / 2C2P สำหรับไทย)
- ระบบ **เช็คสิทธิ์ตาม tier** (feature gating) ฝั่ง backend + UI
- หน้าจัดการแพ็กเกจ / ประวัติการจ่ายเงิน
- ขยาย schema: เพิ่ม `subscription`, `plan`, `wallet/account` ในฐานข้อมูล

### Phase 1 — ปลดล็อก Plus (รายได้ก้อนแรก)
*ฟีเจอร์ที่ผู้ใช้จริงจังยอมจ่าย ทำได้ด้วยทีมเล็ก*

- หลายบัญชี/กระเป๋า (multi-wallet)
- รายจ่ายแบบ recurring + แจ้งเตือน
- รายงานเชิงลึก (กราฟเทรนด์, เปรียบเทียบช่วงเวลา, breakdown ตามหมวด)
- Export Excel / CSV / PDF
- แนบรูปใบเสร็จ + สำรอง/กู้คืนข้อมูล

### Phase 2 — หมัดเด็ด Pro (สร้างความต่างในตลาดไทย)
*ฟีเจอร์ lock-in สูง ทำให้ราคา Pro มีน้ำหนัก*

- **Import statement ธนาคารไทย** (เริ่มจาก CSV/Excel ที่ธนาคารดาวน์โหลดได้ → ต่อยอดสู่ open banking)
- รองรับ **PromptPay / slip** (อ่านสลิปโอนเงิน)
- **OCR ใบเสร็จ** กรอกยอด/ร้าน/วันที่อัตโนมัติ
- **AI จัดหมวดหมู่อัตโนมัติ** จากคำอธิบายรายการ
- รายงานสรุปภาษี

### Phase 3 — ขยายฐานและรายได้เสริม
*หลังมีผู้ใช้จ่ายเงินเป็นกลุ่มแล้ว*

- เป้าหมายการออม + ระบบแจ้งเตือนอัจฉริยะ
- แอปมือถือ (PWA หรือ native) — สำคัญมากสำหรับ B2C การเงิน
- ระบบ **affiliate** แนะนำบัตรเครดิต/ประกัน/กองทุน (รายได้นอกเหนือ subscription)
- แชร์งบในครอบครัว / กลุ่ม

## 6. ลำดับการลงมือที่แนะนำ

1. **Phase 0 ก่อนเสมอ** — สร้างระบบ billing + feature gating ให้พร้อม
2. เปิดตัวด้วย **Free + Plus** เท่านั้น เพื่อพิสูจน์ว่ามีคนยอมจ่าย (validate)
3. เก็บ feedback → สร้าง **Pro (Phase 2)** เป็นจุดขายต่อยอด
4. วัด metric สำคัญ: **conversion rate Free→Paid** (เป้าเริ่มต้น 2–5%), **churn รายเดือน** (เป้า <5%), **ARPU**
5. ค่อยขยายสู่ Phase 3 เมื่อฐานผู้ใช้จ่ายเงินนิ่งแล้ว

---

## 7. ข้อควรระวัง

- **อย่าจำกัด Free เกินไป** จนแอปดูใช้ไม่ได้ — Free คือช่องทาง acquisition ที่ถูกที่สุด
- **Import ธนาคารมีต้นทุน/ความเสี่ยงด้านความปลอดภัยสูง** — เริ่มจาก import ไฟล์ก่อน ค่อยทำ open banking เมื่อพร้อมเรื่อง compliance
- การเก็บข้อมูลการเงินต้องคำนึงถึง **PDPA** ตั้งแต่ออกแบบ
- ทดสอบราคาจริงกับผู้ใช้กลุ่มเล็กก่อนล็อกราคา
