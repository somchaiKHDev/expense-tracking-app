# ข้อกำหนดระบบ (System Specification)
## ระบบบันทึกค่าใช้จ่าย (Expense Tracking System)

---

### 1. บทนำและวัตถุประสงค์ (Introduction & Objective)
เอกสารฉบับนี้จัดทำขึ้นเพื่อกำหนดข้อมูลทางเทคนิคและข้อกำหนดของระบบ (System Specification) สำหรับ **ระบบบันทึกค่าใช้จ่าย (Expense Tracking System)** เพื่อให้นักพัฒนาสามารถนำไปใช้เป็นแนวทางในการพัฒนาต่อทั้งในส่วนของฐานข้อมูล (Backend) และส่วนติดต่อผู้ใช้งาน (Frontend) โดยครอบคลุมตั้งแต่การจัดการรายการค่าใช้จ่ายพื้นฐาน ฟังก์ชันอำนวยความสะดวกอย่างการใช้คำนำหน้ารายการ (Prefix) ไปจนถึงระบบรายงานผลและสรุปยอดค่าใช้จ่าย (Dashboard & Reporting) ตามช่วงเวลาต่างๆ

---

### 2. ข้อกำหนดเชิงฟังก์ชัน (Functional Requirements)

ระบบต้องรองรับการทำงานพื้นฐานและฟังก์ชันหลักดังต่อไปนี้:

#### 2.1 การจัดการรายการค่าใช้จ่าย (Expense CRUD Operations)
* **การเพิ่มรายการ (Create):** ผู้ใช้สามารถกรอกรายละเอียดรายการ เลือกหมวดหมู่ และระบุจำนวนเงินเพื่อบันทึกค่าใช้จ่ายใหม่ได้
* **การแสดงผลรายการ (Read):** แสดงผลข้อมูลค่าใช้จ่ายในรูปแบบตาราง (Data Grid) โดยดึงข้อมูลมาแสดงผลได้อย่างถูกต้องตามโครงสร้าง
* **การแก้ไขรายการ (Update):** ผู้ใช้สามารถแก้ไขข้อมูลในแถว (Row) นั้นๆ ได้ ในกรณีที่เกิดความผิดพลาดในการกรอกข้อมูล
* **การลบรายการ (Delete):** ผู้ใช้สามารถลบแถวรายการค่าใช้จ่ายที่ไม่ต้องการออกจากระบบได้

#### 2.2 ฟังก์ชันการเรียงลำดับและการกรองข้อมูล (Sorting & Filtering)
* **การเรียงลำดับ (Sorting):** 
  * ผู้ใช้สามารถคลิกที่หัวคอลัมน์ **รายการ** เพื่อเรียงตามตัวอักษร (ก-ฮ หรือ A-Z)
  * ผู้ใช้สามารถคลิกที่หัวคอลัมน์ **ประเภท** เพื่อเรียงตามกลุ่มหมวดหมู่
  * ผู้ใช้สามารถคลิกที่หัวคอลัมน์ **ค่าใช้จ่าย** เพื่อเรียงลำดับจากน้อยไปมาก หรือมากไปน้อยได้
* **การกรองข้อมูล (Filtering):** 
  * ผู้ใช้สามารถเลือกกรองข้อมูลเพื่อดูเฉพาะ "ประเภท" หรือ "หมวดหมู่" ที่ต้องการได้ เช่น เลือกแสดงเฉพาะประเภท `วัตถุดิบอาหาร`

#### 2.3 การจัดการหมวดหมู่ (Category Management)
* ระบบต้องมีระบบจัดการหมวดหมู่พื้นฐาน (Pre-defined Categories) เช่น *ครอบครัว, วัตถุดิบอาหาร, ของใช้, อาหารสัตว์, เครื่องสำอาง*
* ระบบควรเปิดให้ผู้ใช้เพิ่ม/แก้ไข/ลบ หมวดหมู่ของตัวเองได้ในอนาคต

#### 2.4 ฟังก์ชันจัดการคำนำหน้ารายการที่ใช้บ่อย (Frequent Item Prefix Management)
เป็นฟังก์ชันที่ช่วยให้ผู้ใช้สามารถเลือก "คำนำหน้า" หรือ "แท็กเริ่มต้น" ที่มีการใช้งานบ่อยมาใส่ในช่องรายละเอียด (Description) ได้ทันที เพื่อลดเวลาในการพิมพ์
* **การบันทึก Prefix อัตโนมัติ (Auto-Suggestion):** ระบบจะคัดเลือกคำที่ผู้ใช้พิมพ์บ่อยที่สุดมาแสดงเป็นตัวเลือก หรือเปิดให้ผู้ใช้ตั้งค่าสร้าง Prefix ล่วงหน้าเองได้
* **การใช้งาน (Action):** เมื่อผู้ใช้คลิกเลือก Prefix ระบบจะนำข้อความนั้นไปใส่ในช่องกรอกรายละเอียด (Description Input Field) ทันที เพื่อให้ผู้ใช้พิมพ์ข้อความต่อท้ายได้สะดวก

#### 2.5 ฟังก์ชันสรุปข้อมูลและรายงานผล (Dashboard & Reporting)
ระบบสามารถประมวลผลข้อมูลจากวันที่บันทึกรายการ (`transaction_date`) เพื่อแสดงผลสรุปยอดค่าใช้จ่ายรวมตามช่วงเวลาที่กำหนดได้ ดังนี้:
* **สรุปรายวัน (Daily Summary):** แสดงยอดรวมค่าใช้จ่ายของวันนี้ และสามารถเปรียบเทียบกับยอดของเมื่อวานได้
* **สรุปรายสัปดาห์ (Weekly Summary):** แสดงยอดรวมค่าใช้จ่ายของสัปดาห์ปัจจุบัน (จันทร์-อาทิตย์) และกราฟแสดงแนวโน้มการใช้จ่ายในแต่ละวันของสัปดาห์
* **สรุปรายเดือน (Monthly Summary):** แสดงยอดรวมค่าใช้จ่ายของเดือนปัจจุบัน พร้อมการแบ่งสัดส่วนค่าใช้จ่ายตามหมวดหมู่ (Category Breakdown)
* **สรุปรายปี (Yearly Summary):** แสดงภาพรวมยอดใช้จ่ายตลอดทั้งปี โดยแยกย่อยเป็นยอดรวมของแต่ละเดือน เพื่อดูพฤติกรรมการใช้เงินในระยะยาว

### 3. โครงสร้างฐานข้อมูล (Database Schema)

แนะนำให้ใช้ฐานข้อมูลเชิงสัมพันธ์ (Relational Database) โดยแบ่งออกเป็น 3 ตารางหลัก:
*(หมายเหตุ: ฟังก์ชันสรุปข้อมูลสามารถใช้คำสั่ง SQL `GROUP BY` ร่วมกับฟิลด์ `transaction_date` ในตาราง `expenses` ได้โดยตรง ไม่จำเป็นต้องสร้างตารางใหม่)*

#### 3.1 ตาราง `categories` (ตารางหมวดหมู่)
| ชื่อฟิลด์ (Field Name) | ประเภทข้อมูล (Data Type) | คุณสมบัติ (Key / Constraints) | คำอธิบาย (Description) |
| :--- | :--- | :--- | :--- |
| `id` | INT | Primary Key, Auto Increment | รหัสประเภทค่าใช้จ่าย |
| `name` | VARCHAR(100) | NOT NULL, UNIQUE | ชื่อประเภท (เช่น ครอบครัว, วัตถุดิบอาหาร) |

#### 3.2 ตาราง `expenses` (ตารางบันทึกค่าใช้จ่าย)
| ชื่อฟิลด์ (Field Name) | ประเภทข้อมูล (Data Type) | คุณสมบัติ (Key / Constraints) | คำอธิบาย (Description) |
| :--- | :--- | :--- | :--- |
| `id` | INT | Primary Key, Auto Increment | รหัสรายการค่าใช้จ่าย |
| `description` | TEXT | NOT NULL | รายละเอียดของรายการ |
| `category_id` | INT | Foreign Key (references `categories.id`) | รหัสประเภทที่เชื่อมโยง |
| `amount` | DECIMAL(10, 2) | NOT NULL | จำนวนเงิน/ค่าใช้จ่าย |
| `transaction_date` | DATE | NOT NULL (Default: Current Date) | วันที่เกิดค่าใช้จ่าย (สำคัญมากสำหรับการทำสรุป) |
| `created_at` | TIMESTAMP | Default: CURRENT_TIMESTAMP | วันเวลาที่บันทึกเข้าสู่ระบบ |

#### 3.3 ตาราง `item_prefixes` (ตารางคำนำหน้าใช้บ่อย)
| ชื่อฟิลด์ (Field Name) | ประเภทข้อมูล (Data Type) | คุณสมบัติ (Key / Constraints) | คำอธิบาย (Description) |
| :--- | :--- | :--- | :--- |
| `id` | INT | Primary Key, Auto Increment | รหัส Prefix |
| `prefix_text` | VARCHAR(50) | NOT NULL, UNIQUE | ข้อความคำนำหน้า (เช่น [ช้อปปิ้ง], [เดลิเวอรี่]) |
| `usage_count` | INT | Default: 0 | จำนวนครั้งที่ถูกเรียกใช้งาน (สำหรับจัดอันดับคำใช้บ่อย) |

### 4. ข้อกำหนดส่วนติดต่อผู้ใช้งาน (UI/UX Specifications)

#### 4.1 การออกแบบหน้าสรุปผล (Dashboard UI)
* **Summary Cards (การ์ดสรุปยอด):** วางไว้ส่วนบนสุดของหน้าจอ ประกอบด้วยการ์ด 4 ใบ ได้แก่:
  1. ยอดรวมวันนี้ (Today)
  2. ยอดรวมสัปดาห์นี้ (This Week)
  3. ยอดรวมเดือนนี้ (This Month)
  4. ยอดรวมปีนี้ (This Year)
* **Charts & Graphs (กราฟแสดงผล):**
  * **Pie Chart / Donut Chart:** สำหรับแสดงสัดส่วนค่าใช้จ่าย "รายเดือน" หรือ "รายปี" แบ่งตามหมวดหมู่ (Category)
  * **Bar Chart / Line Chart:** สำหรับแสดงแนวโน้มค่าใช้จ่าย เช่น กราฟแท่งแสดงยอดใช้จ่าย 7 วันย้อนหลัง หรือ 12 เดือนย้อนหลัง
* **Date Range Picker (ตัวเลือกช่วงเวลา):** มีปุ่มสำหรับเลือกดูรายงานย้อนหลังตามช่วงเวลาที่กำหนดเองได้ (Custom Date Range)

#### 4.2 การออกแบบตารางแสดงผล (Data Grid)
* **ชุดสี (Color Palette):**
  * **Header ของตาราง:** ใช้สีเขียวเข้ม (Forest Green) รหัสสีแนะนำ `#2A5A43` ตัวอักษรสีขาว
  * **แถวข้อมูล (Rows):** ใช้รูปแบบ Zebra Striping (แถวสลับสี) ระหว่างสีขาว `#FFFFFF` และสีเทาอ่อน `#F8F9FA`
* **การจัดวางตำแหน่งข้อมูล (Text Alignment):**
  * คอลัมน์ **รายการ (Description):** จัดชิดซ้าย (Left Aligned)
  * คอลัมน์ **ประเภท (Category):** จัดชิดซ้าย หรือกึ่งกลาง
  * คอลัมน์ **ค่าใช้จ่าย (Amount):** **จัดชิดขวา (Right Aligned)** เสมอ
* **องค์ประกอบเสริม:** หัวตารางต้องมีไอคอนลูกศร (Dropdown/Arrow Icon) สำหรับฟังก์ชัน Sort/Filter และมี **แถวสรุปผลรวม (Footer Row)** ท้ายตาราง

#### 4.3 การออกแบบฟอร์มเพิ่มรายการ (Add Expense Form & Prefix UI)
* **ตัวเลือกคำนำหน้า (Prefix Chips):** เหนือช่องกรอก **"รายการ"** ให้แสดงปุ่มขนาดเล็ก (Chips/Badges) สำหรับ Prefix ที่ใช้บ่อย (เช่น `[วัตถุดิบ]` `[ของใช้บ้าน]`)
* **ขั้นตอนการทำงาน (Workflow):**
  1. ผู้ใช้กดปุ่มเพิ่มรายการค่าใช้จ่าย
  2. เลือกระบุ Prefix โดยคลิกที่ Chip เช่น `[วัตถุดิบ]`
  3. ระบบจะเติมคำว่า `[วัตถุดิบ] ` ลงในช่องรายการอัตโนมัติ และย้าย Cursor ไปไว้ท้ายข้อความ
  4. ผู้ใช้พิมพ์รายละเอียดต่อ เช่น "หมูสับ, เส้นใหญ่" จะได้เป็น `[วัตถุดิบ] หมูสับ, เส้นใหญ่` และกดบันทึก

### 5. ข้อกำหนดการเชื่อมต่อระบบ (API Endpoints)

#### 5.1 API สำหรับรายการและหมวดหมู่ (Expense & Category API)
* `GET /api/v1/expenses` : ดึงข้อมูลรายการค่าใช้จ่าย (รองรับ query params `startDate`, `endDate`, `category`)
* `POST /api/v1/expenses` : สร้างรายการค่าใช้จ่ายใหม่
* `PUT /api/v1/expenses/:id` : อัปเดตหรือแก้ไขรายละเอียดของรายการค่าใช้จ่าย
* `DELETE /api/v1/expenses/:id` : ลบรายการค่าใช้จ่าย
* `GET /api/v1/categories` : ดึงข้อมูลหมวดหมู่ทั้งหมดสำหรับแสดงผลใน Dropdown

#### 5.2 API สำหรับสรุปข้อมูล (Dashboard & Summary API)
* `GET /api/v1/expenses/summary` : ดึงข้อมูลยอดสรุปรวม โดยส่ง Query Parameter `?period=daily|weekly|monthly|yearly`
* `GET /api/v1/expenses/summary/category` : ดึงข้อมูลสรุปแยกตามหมวดหมู่ สำหรับทำกราฟ Pie Chart (รองรับการกรองด้วย `?period=monthly`)

#### 5.3 API สำหรับคำนำหน้า (Prefix API)
* `GET /api/v1/frequent-prefixes` : ดึงรายการคำนำหน้า (Prefix) ที่ใช้บ่อยที่สุดเพื่อแสดงในส่วน Chips
* `POST /api/v1/frequent-prefixes` : สร้างหรือบันทึกคำนำหน้าโปรด (Favorite Prefix)
* `PUT /api/v1/frequent-prefixes/:id/increment` : อัปเดต `usage_count` เมื่อผู้ใช้เลือกใช้งาน Prefix นั้นๆ

### 6. ฟังก์ชันแนะนำเพิ่มเติมเพื่อความสมบูรณ์ (Recommended Enhancements)
1. **ระบบส่งออกรายงาน (Export Report):** สามารถ Export ข้อมูลรายงานรายเดือน/รายปี ออกมาเป็นไฟล์ Excel (.xlsx) หรือ PDF ได้
2. **ระบบแบ่งหน้า (Pagination):** แบ่งหน้าในส่วนของ Data Grid เพื่อลดเวลาการโหลดข้อมูลในกรณีที่มีรายการจำนวนมาก
3. **ฟังก์ชันค้นหา (Search Bar):** ช่องค้นหาสำหรับกรองคำในคอลัมน์ "รายการ" ได้ทันที
